# Pav It — ACF Bootstrap: автоматичне налаштування бекенду

Це заміняє ручне клацання по CPT UI + ACF з попередньої інструкції.
Плагін `pavit-acf-bootstrap.zip` сам:

1. Реєструє всі 8 типів записів (`pav_faq`, `pav_team`, `pav_pricing`, `pav_feature`,
   `pav_stat`, `pav_logo`, `pav_testimonial`, `pav_settings`) з увімкненим REST API —
   CPT UI для цього більше не потрібен (можна лишити активним, він просто не заважає).
2. Реєструє всі ACF-поля кодом (`acf_add_local_field_group`) — точно ті, що описані
   в попередній інструкції, з правильними Return Format / New Lines одразу.
3. При активації одразу створює **реальні записи з реальними значеннями** —
   3 FAQ, 4 учасники команди, 3 тарифи, 10 карток-можливостей, 4 показники
   статистики, 5 логотипів клієнтів, 1 відгук, і один запис "Налаштування сайту"
   з усіма ~50 текстовими полями сайту.

## Встановлення

1. **Спочатку переконайтесь, що ACF активний** (Plugins → Advanced Custom Fields → Активувати).
2. Розпакуйте `pavit-acf-bootstrap` у `wp-content/plugins/` вашого `pav-it-be`.
3. Plugins → знайдіть **"Pav It — ACF Bootstrap"** → Активувати.
   - Якщо побачите помилку "requires 1 plugin... advanced-custom-fields" — значить ACF
     не активний, поверніться до кроку 1.
4. Готово. Зліва в меню з'явиться **"Контент сайту"** з усіма розділами, уже заповненими.

## Перевірка

```
http://localhost/pav-it-be/wp-json/wp/v2/pav-faq
http://localhost/pav-it-be/wp-json/wp/v2/pav-settings
```

Мають повернути JSON з реальним текстом і об'єктом `acf` у кожному записі.

## Що робити далі

- **Редагувати значення** — заходите в потрібний розділ ("Команда", "Тарифи" тощо),
  відкриваєте запис, міняєте текст у звичних ACF-полях під заголовком, зберігаєте.
- **Додати новий запис** (напр. ще одного учасника команди) — просто "Додати новий"
  у відповідному розділі, заповнити поля так само, як у решти записів.
- Повторна активація плагіна **не** створює дублікати — засів контенту відбувається
  лише один раз, якщо в типі запису вже є хоч один опублікований запис.

## Якщо хочете все ж керувати через CPT UI/ACF вручну

Попередня інструкція (`README-ACF-SETUP.md`) з таблицями полів і кроками для
ручного налаштування через інтерфейс лишається чинною — просто не активуйте
цей плагін і зробіть все власноруч за тими таблицями.
